<Tip>
  Start with `feature_miner.py` before running the full ensemble. It runs in seconds and immediately demonstrates how arithmetic dimensions reveal prime structure.
</Tip>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ShanmuganandK/Primenumber_nueral_network.git
    cd Primenumber_nueral_network
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install numpy pandas scikit-learn sympy
    ```

    The four libraries each serve a specific role:

    | Package | Purpose |
    |:---|:---|
    | `numpy` | Array operations and binary representation utilities |
    | `pandas` | DataFrame construction for the multi-dimensional dataset |
    | `scikit-learn` | `DecisionTreeClassifier` (Model A) and `MLPClassifier` (Model B) |
    | `sympy` | `isprime()` — the ground-truth oracle for labeling training data |
  </Step>

  <Step title="Run the basic feature miner">
    The feature miner analyzes integers 1 through 100 and prints the arithmetic rules it discovers:

    ```bash
    python feature_miner.py
    ```

    Expected output:

    ```text
    Extracting dimensions for numbers 1 to 100...

    Total Primes: 25
    Total Composites: 74

    --- POSITIVE RULES FOR COMPOSITES ---
    Rule 1 [is_even and > 2]: Captures 49 composite numbers.
    Rule 2 [ends_with 5 and > 5]: Captures 9 composite numbers.
    Rule 3 [digit_root in (3,6,9) and > 3]: Captures 30 composite numbers.

    Composites NOT covered by these 3 rules: [49, 77, 91]
    Notice anything? These are all multiples of 7: [49, 77, 91]

    --- OBSERVATIONS ON PRIMES ---
    Primes > 5 ONLY end in digits: [1, 3, 7, 9]
    Primes > 3 ONLY have digit roots: [1, 2, 4, 5, 7, 8] (Never 3, 6, or 9)
    ```

    The three composite rules eliminate most numbers immediately. The three survivors — `49`, `77`, `91` — are all multiples of 7, demonstrating exactly why the `mod_7` dimension is required as the next filter stage.
  </Step>

  <Step title="Run the full two-stage ensemble">
    ```bash
    python neural_net.py
    ```

    This script builds and evaluates the two-stage pipeline:

    1. **Generates data** for integers 1 – 20,000, encoding each number using `extract_all_dimensions()`.
    2. **Trains Model A** — a shallow `DecisionTreeClassifier` (max depth 5) on base-system filter features: `last_digit_base6`, `last_digit_base30`, digit sums, and even-length palindrome flags. Model A eliminates the roughly 70% of integers that are obviously composite.
    3. **Trains Model B** — a `MLPClassifier` with hidden layers `(64, 128, 64)` and ReLU activations on the candidates that survived Model A (the "intersections"). Model B uses complex features: binary ratios, `distance_from_max_sum`, and `last_digit_base210`.
    4. **Evaluates the ensemble** on the completely unseen range 10,001 – 20,000, reporting overall accuracy and a full classification report broken down by Composite and Prime.
  </Step>

  <Step title="Run the comprehensive evaluation">
    ```bash
    python final_ensemble_test.py
    ```

    This script re-trains the ensemble on 1 – 10,000 and evaluates it against three unseen ranges:

    - **10,001 – 20,000**: near-range extrapolation
    - **50,000 – 60,000**: mid-range extrapolation
    - **100,000 – 110,000**: far-range extrapolation

    The final ensemble uses a stricter Model A configuration — `class_weight={0: 20, 1: 1}` to heavily penalize false positives — and a high-confidence threshold of `0.9` on Model B's probability output before labeling any number as prime.

    <Note>
      `final_ensemble_test.py` generates feature vectors for approximately 130,000 numbers, including Fermat witness calculations via modular exponentiation. Expect a runtime of several minutes depending on your machine.
    </Note>
  </Step>

  <Step title="Extract dimensions for a single number">
    You can call the encoder directly to inspect the full feature vector for any integer:

    ```python
    from neural_encoder import extract_all_dimensions

    dims = extract_all_dimensions(17)
    for k, v in dims.items():
        print(f"{k}: {v}")
    ```

    - `is_rem_30_prime: 1` — the remainder `17 mod 30 = 17` is itself prime, activating the Prime Feedback Effect.
    - `rem_30_base3_sum: 5` — the Base-3 digit sum of that remainder is exactly 5, activating the +150% Fractal Geometry signal.
  </Step>
