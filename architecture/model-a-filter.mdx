---
title: "Model A: Base-System Filter"
description: "The Decision Tree that eliminates ~77% of composites using rigid mathematical rules."
---

Model A is the first stage of the ensemble. Its job is fast, safe elimination of composites whose non-primality can be proven from base-system arithmetic alone — without any non-linear reasoning.

## Algorithm

```python
from sklearn.tree import DecisionTreeClassifier

model_a = DecisionTreeClassifier(
    max_depth=5,
    class_weight={0: 20, 1: 1},
    random_state=42
)
model_a.fit(train_df[filter_features], train_df['is_prime'])
```

<Note>
  `max_depth=5` is deliberately shallow to prevent overfitting. Model A is designed to learn only the bold, provable mathematical laws encoded in base-system last digits and indivisibility flags — not memorised patterns.
</Note>

## Feature set

```python
filter_features = [
    'last_digit_base6', 'last_digit_base30',
    'is_indivisible_by_2', 'is_indivisible_by_3',
    'is_indivisible_by_5', 'is_indivisible_by_7'
]
```

These six features encode mathematically provable rules:

| Feature | Mathematical law encoded |
|:---|:---|
| `last_digit_base6` | All primes > 5 must end in `1` or `5` in Base-6. Endings `0, 2, 3, 4` are divisible by 2 or 3. Eliminates **66%** of integers. |
| `last_digit_base30` | All primes > 30 must end in one of 8 digits: `[1, 7, 11, 13, 17, 19, 23, 29]`. Eliminates **73.3%** of integers. |
| `is_indivisible_by_2` | `1` if `n % 2 ≠ 0` (or `n == 2`), else `0` |
| `is_indivisible_by_3` | `1` if `n % 3 ≠ 0` (or `n == 3`), else `0` |
| `is_indivisible_by_5` | `1` if `n % 5 ≠ 0` (or `n == 5`), else `0` |
| `is_indivisible_by_7` | `1` if `n % 7 ≠ 0` (or `n == 7`), else `0` |

The indivisibility flags are extracted in `neural_encoder.py`:

```python
dimensions['is_indivisible_by_2'] = int(n % 2 != 0 or n == 2)
dimensions['is_indivisible_by_3'] = int(n % 3 != 0 or n == 3)
dimensions['is_indivisible_by_5'] = int(n % 5 != 0 or n == 5)
dimensions['is_indivisible_by_7'] = int(n % 7 != 0 or n == 7)
```

The small primes (2, 3, 5, 7) are explicitly treated as indivisible by themselves, so they are never incorrectly pruned.

## Class weighting

The `{0: 20, 1: 1}` class weight means the training loss treats every false positive (a composite labelled prime) as **20 times worse** than a false negative (a prime labelled composite). This asymmetry is intentional:

- **False positive** (composite → prime): catastrophic — contaminates Model B's input with known-composite numbers.
- **False negative** (prime → composite): recoverable — the prime is simply never considered, reducing recall but not precision.

The result is a filter with **zero false positives** consistently observed across all evaluation ranges.

## Probability threshold

Model A does not use a hard `predict()` call. Instead it uses `predict_proba()` with a soft threshold to avoid over-confident elimination:

```python
train_proba_a = model_a.predict_proba(train_df[filter_features])[:, 1]
train_df['model_a_pred'] = (train_proba_a > 0.05).astype(int)
```

During training, numbers with P(prime) > 0.05 are forwarded to Model B as intersection candidates. At inference time, the threshold shifts to 0.01, reflecting the heavier class-0 weighting applied during training:

```python
unsure_mask = probs_a >= 0.01
```

Only numbers below this threshold are discarded as composite. Everything uncertain is passed forward.

## Performance

Model A consistently prunes approximately **77% of composites** across every evaluated range:

| Range | Model A Pruning |
|:---|:---|
| 1 – 10,000 (train) | 77.11% |
| 10,001 – 20,000 | 77.01% |
| 100,000 – 110,000 | 77.04% |

The stability of this figure across 100k+ numbers demonstrates that the base-system rules are genuinely scale-invariant — not dependent on magnitude.
