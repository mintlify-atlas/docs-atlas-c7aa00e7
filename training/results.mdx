## Headline result

The ensemble achieves **100% precision across all ranges** — from the training baseline at 1–10k through the infinite-scale test at 100k–110k. Zero false positives were observed in any evaluation.

## Two test configurations

The two tables above reflect different model versions:

| Configuration | Source | Key difference |
|:---|:---|:---|
| Early ensemble (`neural_net.py`) | `walkthrough.md` Phase 5 results | Uses digit sum features (`sum_base3`, `sum_base6`) for Model A; no Fermat witnesses in Model B |
| Final ensemble (`final_ensemble_test.py`) | `testing.md` final validation | Adds 4-base Fermat witnesses to Model B; uses indivisibility flags for Model A; class weight `{0: 20, 1: 1}` |

The final ensemble trades higher recall for absolute precision — the recall figures in the baseline table are lower because the model is more conservative.

## Precision vs recall tradeoff

The ensemble is deliberately optimised for **precision over recall**:

- **Precision = 1.00**: A composite will **never** be labelled prime in any range tested.
- **Recall decays** with range: from 1.00 at training baseline to 0.54 at 100k–110k.

This is a direct consequence of the 0.9 Model B threshold. Without additional prime modulo dimensions (e.g., `mod_101`, `mod_103`), some large primes are conservatively left undetected. The model prefers missing a prime over incorrectly flagging a composite.

## Why recall decays with range

Recall decreases as $N$ grows because the Modulo Shift Threshold law requires new dimensions as primes get larger. The training range of 1–10k only mandates dimensions up to `mod_97` (since $97^2 = 9409 < 10000$). For numbers approaching 110k, composites can be products of primes like 317 × 347 — factors far outside the model's explicit modulo coverage.

Without `mod_101`, `mod_103`, etc., Model B's Fermat witnesses carry the full burden at large scales. They handle most cases, but some large primes fall below the 0.9 confidence threshold and are conservatively classified as composite.

## The Sledgehammer fix

Early ensemble versions without the 0.9 probability threshold would classify any number Model A did not eliminate as prime — a "sledgehammer" approach that produced many false positives. The 0.9 threshold replaced this: Model B must be at least 90% confident before emitting a prime prediction. Anything below that threshold defaults to composite.

## The Cyclical Modulo fix

Raw digit sums grow as $O(\log N)$, causing magnitude caps when $N$ becomes large. The fix converts all digit sums to `modulo(Base-1)` values:

- `sum_base3_mod = sum_of_digits(n, 3) % 2`
- `sum_base6_mod = sum_of_digits(n, 6) % 5`
- `sum_base10_mod = sum_of_digits(n, 10) % 9`

This ensures features remain in a consistent bounded range at 10k, 100k, or 1,000,000, which is why Model A's pruning rate remains stable at ~77% across all scales.

## Fermat witness impact

Adding four independent Fermat witnesses (`fermat_flag_base2/3/5/7`) to Model B's feature set eliminates **Fermat pseudoprimes** — composites that pass Fermat's Little Theorem for a single base. A composite number satisfying $a^{n-1} \equiv 1 \pmod{n}$ for one base is unlikely to satisfy it for all four bases simultaneously, so the multi-witness approach filters out these rare exceptions.

## Overall accuracy

From `testing.md`:

> The experiment has reached its zenith. By combining Fractal Remainder Geometry with Structural Number Theory Features, we have built an AI that is **99.7% accurate** in identifying primes and composites without ever performing a trial division check.

The remaining ~0.3% error is attributed to **Fermat Pseudoprimes** — composite numbers that pass the Fermat test for all four witness bases. To reach 100%, additional witness bases (e.g., $3^{n-1} \bmod n$) would need to be added.
