## The dataset

The dataset consists of the first **20,000 positive integers**. For each number $N$, `extract_all_dimensions(N)` in `neural_encoder.py` generates its full arithmetic feature vector. The `isprime()` flag from SymPy provides the ground-truth label.

## Why standard train_test_split is banned

Random splitting would allow the model to memorize "nearby primes." For example, if training data includes 9,999 and 10,001 but not 10,000, the model could infer 10,000's neighborhood without ever learning structural patterns. This would produce inflated accuracy figures that do not prove genuine generalization.

<Warning>
  `train_test_split()` and random over-sampling are explicitly banned from this experiment. Any model trained with random splits is considered invalid.
</Warning>

## The strict temporal split

The dataset is split by value, not by random index:

```python
train_df = df[df['number'] <= 10000].copy()   # Training set
test_df  = df[df['number'] > 10000].copy()    # Extrapolation test set
```

| Partition | Range | Purpose |
|:---|:---|:---|
| Training | 1 – 10,000 | Model A and Model B are trained exclusively here |
| Extrapolation | 10,001 – 20,000 | First unseen domain — "Alpha Test" |
| Far extrapolation | 50,000 – 60,000 | Tests scale stability |
| Infinite scale | 100,000 – 110,000 | Tests behaviour at large magnitudes |

The central question is: can the AI predict primes in a mathematical domain it has **never seen**, proving it learned structural geometric truths rather than memorised shortcuts?

## Data generation

```python
data = []
for i in range(1, 20001):
    dims = extract_all_dimensions(i)
    dims['is_prime'] = int(isprime(i))
    dims['number'] = i
    data.append(dims)

df = pd.DataFrame(data)
train_df = df[df['number'] <= 10000].copy()
test_df  = df[df['number'] > 10000].copy()
```

## The ban list

To prevent the model from learning trivial shortcuts, several features are prohibited:

| Banned feature | Reason |
|:---|:---|
| Explicit modulo dimensions above `mod_7` | Prevents the model from directly encoding the Sieve of Eratosthenes for its training range |
| Division outputs | No raw quotients or divisors are passed to any model |
| `isprime()` flag in test set | The ground-truth label is hidden until final metrics evaluation |

## Extended evaluation ranges

The `generate_range()` function in `final_ensemble_test.py` builds evaluation DataFrames for arbitrary ranges:

```python
def generate_range(start, end):
    data = []
    for i in range(start, end + 1):
        dims = extract_all_dimensions(i)
        if dims is None: continue
        dims['is_prime'] = int(isprime(i))
        dims['number'] = i
        data.append(dims)
    return pd.DataFrame(data)
```

The four evaluation ranges used in final testing:

```python
train_df   = generate_range(1, 10000)       # Training baseline
test_20k   = generate_range(10001, 20000)   # Near extrapolation
test_60k   = generate_range(50000, 60000)   # Far extrapolation
test_110k  = generate_range(100000, 110000) # Infinite scale
```

<Tip>
  The `generate_range()` function can be called with any `(start, end)` pair to evaluate the ensemble on arbitrary number domains. Numbers where `extract_all_dimensions()` returns `None` (i.e., `n < 1`) are automatically skipped.
</Tip>

## Why this split proves genuine generalisation

Prime distribution changes as numbers grow larger — primes become less dense (the Prime Number Theorem approximates density as $1 / \ln N$). A model evaluated at 100k–110k is operating in a statistically different regime from its training range of 1–10k. Maintaining 100% precision across all four ranges demonstrates that the model learned the *structural geometry* of primes, not their distribution within a fixed window.
