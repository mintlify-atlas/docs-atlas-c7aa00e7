The `feature_miner` module is the earliest-stage exploration tool in the pipeline. It computes a compact set of arithmetic dimensions for any integer and applies human-readable composite rules to reveal structural patterns in the primes up to 100.

## Functions

### `generate_dimensions(n)`

Extracts a compact set of arithmetic dimensions for a single integer.

<ParamField path="n" type="int" required>
  The positive integer to analyze.
</ParamField>

**Returns:** `dict` with the following keys:

<ResponseField name="number" type="int">
  The input integer `n` itself.
</ResponseField>

<ResponseField name="is_prime" type="bool">
  `True` if `n` is prime, determined by trial division (see `is_prime` below).
</ResponseField>

<ResponseField name="is_even" type="bool">
  `True` if `n % 2 == 0`.
</ResponseField>

<ResponseField name="ends_with" type="int">
  `n % 10` â€” the last digit in Base-10.
</ResponseField>

<ResponseField name="digit_sum" type="int">
  Sum of all Base-10 digits of `n`.
</ResponseField>

<ResponseField name="digit_root" type="int">
  The single-digit result of repeatedly summing the digits of `n`. Returns `9` for any multiple of 9 (rather than 0).
</ResponseField>

<ResponseField name="num_digits" type="int">
  Number of decimal digits in `n` (`len(str(n))`).
</ResponseField>

<ResponseField name="is_perfect_square" type="bool">
  `True` if `math.isqrt(n) ** 2 == n`.
</ResponseField>

<ResponseField name="mod_3" type="int">
  `n % 3`.
</ResponseField>

<ResponseField name="mod_5" type="int">
  `n % 5`.
</ResponseField>

<ResponseField name="mod_7" type="int">
  `n % 7`.
</ResponseField>

#### Source

```python
def generate_dimensions(n):
    dimensions = {
        'number': n,
        'is_prime': is_prime(n),
        'is_even': n % 2 == 0,
        'ends_with': n % 10,
        'digit_sum': digit_sum(n),
        'digit_root': digit_root(n),
        'num_digits': len(str(n)),
        'is_perfect_square': math.isqrt(n)**2 == n,
        'mod_3': n % 3,
        'mod_5': n % 5,
        'mod_7': n % 7
    }
    return dimensions
```

### `analyze_1_to_100()`

Runs early-phase rule mining on integers 1â€“100, printing composite-detection rules and uncovered composites to stdout.

**Parameters:** None

**Returns:** `None`

#### What it does

Generates dimensions for every integer from 1 to 100, then applies three simple explicit rules to identify composites:

| Rule | Condition | Composites captured |
|------|-----------|---------------------|
| Rule 1 | `is_even` and `number > 2` | All even composites |
| Rule 2 | `ends_with == 5` and `number > 5` | Multiples of 5 ending in 5 |
| Rule 3 | `digit_root in {3, 6, 9}` and `number > 3` | All composites divisible by 3 |

After applying all three rules, the composites that remain uncovered are all multiples of 7 (49, 77, 91) â€” numbers that are not divisible by 2, 3, or 5.

<Tip>
  The output also prints an observation about primes: every prime greater than 5 ends only in the digits `{1, 3, 7, 9}`, and every prime greater than 3 has a digit root in `{1, 2, 4, 5, 7, 8}` â€” never 3, 6, or 9.
</Tip>

#### How to run

```bash
python feature_miner.py
```

### Utility functions

These helpers are used internally by `generate_dimensions` and are not intended to be called directly by downstream pipeline modules.

#### `is_prime(n)`

Trial-division primality test. Does not use `sympy`.

- Returns `False` for `n â‰¤ 1`.
- Returns `True` for `n â‰¤ 3`.
- Checks divisibility by 2 and 3, then iterates candidate factors of the form `6k Â± 1` up to `âˆšn`.

#### `digit_sum(n)`

Returns the sum of all Base-10 digits of `n` (`sum(int(d) for d in str(n))`).

#### `digit_root(n)`

Iteratively sums the digits of `n` until a single digit remains. Returns `9` for any multiple of 9 (instead of 0), matching the conventional digital root definition.

```python
def digit_root(n):
    if n == 0:
        return 0
    return 9 if n % 9 == 0 else n % 9
```
