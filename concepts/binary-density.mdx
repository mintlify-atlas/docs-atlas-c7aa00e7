## The Discovery

A statistical analysis of the first 100,000 integers revealed a measurable structural difference between primes and composites in binary:

| Number type | Average fraction of 1-bits |
|-------------|---------------------------|
| Composites | 51.7% |
| Primes | **55.4%** |

Primes are statistically "heavier" in binary than composites. This is not a rounding artifact — it is a consistent signal that persists across all tested ranges.

## Why This Works

The more 1-bits a number has in binary, the more terms appear in its binary expansion, and the harder it becomes to factor cleanly. A number with high binary density tends to resist simple divisibility patterns — its factors, if they exist, must be larger primes.

This is confirmed by the correlation between binary 1-count and the average prime factor required to break the number:

| Binary ones count | Average mod factor required |
|-------------------|---------------------------------|
| 2 | mod 2.24 |
| 5 | mod 5.85 |
| 10 | mod 12.25 |

Numbers with only 2 ones in binary (like powers of 2 plus one: 3, 5, 9, 17...) are typically caught by very small prime factors. Numbers with 10 or more binary ones require factors in the range of `mod_11` or higher to detect, placing them in the harder region of the classification problem.

<Info>
Binary population count (popcount) is a standard CPU instruction on modern hardware. Computing `bin(n).count('1')` is extremely fast — this feature adds almost no computational cost to the feature extraction pipeline.
</Info>

## First Digit: Zero Signal

As a contrast, the project also tested whether the **first decimal digit** of N carries any information about which prime factor is required to catch it.

Result: the first digit has virtually **zero impact**. Across numbers starting with any digit from 1 to 9, the distribution of required mod dimensions is nearly uniform:

- ~55% require `mod_2`
- ~18% require `mod_3`
- ~7.4% require `mod_5`

This holds regardless of whether the number begins with 1, 5, or 9. The leading digit encodes magnitude position (Benford's law territory), not prime structure.

## Palindromes: A Structural Anomaly

Palindromic numbers in base 10 (numbers that read the same forwards and backwards, e.g., 101, 1221, 4554) behave differently from non-palindromes:

| Category | Caught by `mod_2` | Rate |
|----------|-------------------|------|
| Normal numbers | 55.4% of composites | Baseline |
| Palindromes | Only 49.5% of composites | Lower |

Palindromes are less likely to be caught by `mod_2` because their digit symmetry biases them toward odd values (an even-digit palindrome must end in the same digit it starts with; many palindromes are odd).

### The Even-Length Palindrome Rule

A stronger structural rule emerges for even-length palindromes (44, 1221, 9889, 123321):

**Every even-length palindrome is always divisible by 11.**

This is because an even-length palindrome `d_1 d_2 … d_k d_k … d_2 d_1` satisfies the alternating digit sum test for 11. The alternating sum (add odd-position digits, subtract even-position digits) is zero for any palindrome with an even number of digits, and a number with alternating digit sum of 0 is divisible by 11.

Therefore `mod_11` is a master rule for the entire category of even-length palindromes — not a probabilistic signal, but a mathematical certainty.

<Warning>
The `is_palindrome_10` feature in the dimension vector is most useful in combination with `len_base10`. A model that sees `is_palindrome_10 = 1` AND `len_base10 = 4` can apply the even-length rule with certainty. Either feature alone is weaker.
</Warning>

## The Ratio Feature

The digit density is captured explicitly in the feature vector:

```python
# From neural_encoder.py
dimensions['ratio_base10'] = sum_of_digits(n, 10) / dimensions['len_base10']
```

This is the **average decimal digit value** — the digit sum divided by digit count. A higher ratio means the digits trend toward 9; a lower ratio means digits trend toward 0 or 1.

- Composite average digit value: **4.596**
- Prime average digit value: **4.672**

The gap is small but consistent. Combined with the binary density signal, the two ratio features give Model B (the DNN) a complementary pair of "weight" measurements — one in base 10, one in base 2.

## Role in the Ensemble

These density features are **complex features** in the sense that they are not simple divisibility rules — they are statistical correlates of primality that no rule-based system can express. As such, they are primarily useful to **Model B (the Deep Neural Network)** rather than Model A (the Decision Tree filter).

Model A relies on sharp, mathematically exact rules (base-30 endings, digit sum divisibility). Model B uses the full 24-dimensional vector including ratio and density features to resolve the ambiguous cases — the "intersections" where Model A's certainty falls below 95%.
